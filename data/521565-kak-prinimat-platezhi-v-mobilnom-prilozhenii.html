Как принимать платежи в мобильном приложении?

Сегодня многие компании разрабатывают собственные приложения для смартфонов. Если речь идет о бизнесе, который продает товары или услуги, то часто возникает вопрос – как принимать оплату в мобильном приложении? О том, как все устроено, рассказываем ниже.

Разбираемся в терминах

Начиная гуглить способы приема платежей в приложении, вы неизбежно столкнетесь с терминами API и Mobile SDK, поэтому давайте разбираться.

API (Application Programming Interface) – это совокупность инструментов, которые помогают использовать функции одного приложения внутри другого.

Mobile SDK (Software Development Kit) – это набор готовых решений для разработки ПО под определенную платформу.

Оба инструмента используются для подключения платежной формы в приложении, однако Mobile SDK подразумевает упрощенный процесс интеграции.

Как подключить прием платежей в мобильном приложении?

Прежде, чем приступить к интеграции платежной формы в приложении, вам понадобится подключить интернет-эквайринг. Сделав это, вы получите доступ к личному кабинету, где сможете отслеживать все транзакции. О том, как подключить интернет-эквайринг, можете прочитать в одной из наших предыдущих статей.

Завершив процесс подключения эквайринга, можно приступать к работе над приложением. На этом этапе у вас будет два возможных пути:

  • Провести подключение с помощью API

На это потребуется больше времени для работы разработчиков и дизайнеров. Однако вы сможете персонализировать платежную форму и оформить ее в соответствии с айдентикой вашего бренда.

  • Интегрировать платежную форму с помощью Mobile SDK

Требует минимальных трудозатрат и времени на интеграцию, потому что вы можете воспользоваться уже готовыми библиотеками и не тратить время на отрисовку платежной формы. Мы все сделали за вас!

Почему Mobile SDK удобнее и проще

Использование Mobile SDK подразумевает интеграцию нативной платежной формы внутри вашего приложения или использование своей формы. В отличие от интеграции с помощью API, на подключение приема платежей с помощью Mobile SDK требуется минимальное количество времени. Все, что от вас нужно, это подключить «библиотеку» в структуру приложения. На этом все, вопрос решен!

При подключении Mobile SDK от CloudPayments, клиент может воспользоваться стандартной платежной формой из-за чего процесс подключения становится максимально простым и происходит почти мгновенно. В то время, как использование API требует дополнительный материальных и временных ресурсов на работу разработчиков и отрисовку платежной формы дизайнерами. Поэтому процесс интеграции нередко затягивается элементарно из-за человеческого фактора.

Кому понадобится Mobile SDK

Наиболее актуальным подключение Mobile SDK будет для владельцев малого, среднего бизнеса или стартапов, которые хотят запуститься быстро без лишних заморочек.

Позже по мере развития бизнеса или желания, вы можете обратиться к API и дополнительным возможностям сервиса: отрисовать собственную платежную форму, согласно айдентике вашей компании, или интегрировать дополнительные возможности платежного сервиса. Это может быть функция «безопасной сделки», рекуррентные платежи, рекарринг или технологию AFT (Account Funding Transaction), которая подойдет кредитным организациям и МФО.

Как принимать платежи с помощью Mobile SDK

  • Клиент формирует заказ на товар или услугу;
  • При покупке выбирает любой удобный способ оплаты: банковской картой, Yandex Pay или СБП (также доступна оплата с помощью Apple Pay или Google Pay в странах СНГ);
  • Вы получаете уведомление об успешной транзакции в личном кабинете CloudPayments;
  • Выводите средства на свой счет или проводите необходимые денежные манипуляции – возврат или выплаты.
